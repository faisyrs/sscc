# SSCC-p00001: Engine Core Properties

**Level**: prd | **Status**: Active | **Implements**: -

**Purpose:** The SSCC engine is a game-agnostic rules engine built around four primitives: State, Sequence, Condition, and Choice. This requirement captures the five high-level behavioral properties that any conforming engine must exhibit. These properties are observable from outside the engine and do not prescribe internal architecture.

## Assertions

A. The engine SHALL derive all game behavior exclusively from external **pack** files containing timeline, glossary, rules, initial state, and manifest data.

B. When the engine uses randomness, the engine SHALL produce identical outcomes given identical inputs and an identical random seed.

C. When the engine offers a choice, blocks a choice, applies an effect, or transitions state, the engine SHALL record which rules fired, which conditions were satisfied, which reason keys changed, and which random values were used.

D. When a new game is introduced, the engine SHALL support it through a new **pack** without modification to the engine itself.

E. When a **pack** contains malformed data, the engine SHALL reject it at load time with diagnostic messages identifying every validation failure.
