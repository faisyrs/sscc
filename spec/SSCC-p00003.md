# SSCC-p00003: Timeline Execution

**Level**: prd | **Status**: Active | **Implements**: -

**Purpose:** The timeline is a finite, parameterized event template that the engine walks deterministically. It is built from exactly five node types: event, sequence, repeat, forEach, and subSequence. Because every node type is finite and bounded, the engine can enumerate all possible event IDs at load time, enabling pre-validation of every rule trigger. The engine also emits system events automatically in response to choice-related state changes.

## Assertions

A. The engine SHALL construct timelines from exactly five node types: event, sequence, repeat, forEach, and subSequence.

B. When the engine walks a timeline, the engine SHALL emit events in the deterministic order defined by the timeline node tree.

C. When a **pack** is loaded, the engine SHALL enumerate all possible event IDs from the timeline at load time.

D. When a **pack** is loaded, the engine SHALL validate every rule trigger event against the set of event IDs enumerable from the timeline and the set of system events.

E. When a choice-related state change occurs, the engine SHALL emit the corresponding system event automatically without any declaration in the **pack** timeline.

F. When a forEach node iterates over an empty set, the engine SHALL produce zero iterations for that node.
