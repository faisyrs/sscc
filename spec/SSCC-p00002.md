# SSCC-p00002: State Management

**Level**: prd | **Status**: Active | **Implements**: -

**Purpose:** State is the complete structured snapshot of facts at any point in time. It includes the active player, round number, entities in play, unit statuses, resources, usage flags, and open reaction windows. State changes only through explicit engine effects, never through side channels. The initial_state.json file in a pack defines both the starting values and the canonical state schema that all path references must resolve against.

## Assertions

A. The engine SHALL represent state as a complete structured snapshot of all game facts at a given point in time.

B. The engine SHALL modify state only through explicit engine effects.

C. The engine SHALL treat the **initial_state.json** file in a loaded **pack** as the canonical definition of the state schema.

D. When a **pack** is loaded, the engine SHALL validate every state path reference in predicates and effects against the state schema defined by **initial_state.json**.

E. If any state path reference in a **pack** cannot resolve against the state schema, the engine SHALL reject the **pack** at load time.
