# SSCC-p00004: Rule Evaluation

**Level**: prd | **Status**: Active | **Implements**: -

**Purpose:** Rules (rulelets) are the mechanism by which pack data drives game behavior. Each rule is event-triggered, condition-guarded, effect-bearing, and priority-ordered. When the engine arrives at an event, it collects matching rules, evaluates their conditions, groups them by conflict domain, and resolves conflicts using one of three strategies: stack, override, or patch. Priority ordering is ascending -- low priority values fire first, high priority values fire last. Conditions are pure boolean functions that read state but never modify it.

## Assertions

A. When the engine arrives at an event, the engine SHALL collect all rules whose trigger event matches the current event.

B. The engine SHALL evaluate each collected rule's conditions as pure boolean functions that produce no side effects.

C. The engine SHALL group matching rules by conflict domain derived from their effects.

D. When multiple rules share the same conflict domain with strategy "stack", the engine SHALL apply all of them.

E. When multiple rules share the same conflict domain with strategy "override", the engine SHALL apply only the rule with the highest priority value.

F. When two rules share the same conflict domain, the same trigger event, strategy "override", and the same priority value, the engine SHALL reject the **pack** at load time.

G. When rules with strategy "patch" exist in a conflict domain, the engine SHALL apply them after stack and override resolution.

H. The engine SHALL execute resolved effects in ascending priority order, where lower numeric priority values fire before higher numeric priority values.
