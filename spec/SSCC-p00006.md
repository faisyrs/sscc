# SSCC-p00006: Data Pack System

**Level**: prd | **Status**: Active | **Implements**: -

**Purpose:** All game behavior is defined in external data pack files
rather than in engine code. A pack contains a manifest, timeline
definition, glossary, rules, and initial state. Packs may declare
dependencies on other packs, with defined load order and merge
behavior. The engine validates all cross-references within and across
packs at load time, rejecting malformed packs with diagnostic messages
before gameplay begins. Selectors provide named, typed, set-valued
queries over game state. Effect verbs have typed schemas that the
engine enforces.

## Assertions

A. The engine SHALL load game behavior exclusively from external pack files.

B. The engine SHALL require each pack to contain a manifest, a timeline definition, a glossary, a rules file, and an initial state file.

C. The engine SHALL validate the manifest for pack identity, version, engine compatibility, and declared dependencies.

D. When a pack declares dependencies, the engine SHALL load those dependencies first in their declared order before loading the declaring pack.

E. When packs are merged, the engine SHALL accept glossary entries that are duplicated with identical definitions across packs.

F. If merged packs contain glossary entries with the same key but conflicting definitions, the engine SHALL reject the merge with a diagnostic message.

G. When packs are merged, the engine SHALL preserve the base pack's timeline structure without allowing dependent packs to replace it.

H. The engine SHALL validate all state path references in predicates and effects against the initial state schema at load time.

I. If a state path reference cannot be resolved against the initial state schema, the engine SHALL reject the pack with a diagnostic message identifying the unresolvable path.

J. The engine SHALL validate that every rule trigger event matches an event declared in the timeline or a recognized system event.

K. Each **selector** SHALL be a named, typed, set-valued query that returns a set of entity identifiers.

L. The engine SHALL validate each selector's entity kind against the set of known entity types at load time.

M. Each **effect verb** SHALL have a typed parameter schema that the engine validates at load time.

N. The engine SHALL validate that every action reference in a choice declaration resolves to an existing rule.

O. The engine SHALL validate that every reason key referenced in a rule is declared in the glossary.
