# SSCC-p00007: Engine API

**Level**: prd | **Status**: Active | **Implements**: -

**Purpose:** The engine exposes a set of functions that allow callers
to load game packs, initialize game state, advance through the timeline,
interact with player choices, inspect blocked choices, and manage state
snapshots. The API also supports forward simulation with a configurable
decision policy. These functions form the primary interface through
which external systems interact with the engine.

## Assertions

A. The engine SHALL expose a function to load, validate, and merge a pack from a given path, returning validation errors if the pack is malformed.

B. The engine SHALL expose a function to create initial game state from a loaded pack, accepting optional state overrides.

C. The engine SHALL expose a function to advance the timeline forward by one event from the current state.

D. The engine SHALL expose a function to enumerate all currently legal choices for a given state and event.

E. The engine SHALL expose a function to apply a player's selected choice to the current state, accepting an optional random seed.

F. The engine SHALL expose a function that, given a proposed choice and current state, returns the reason keys and rule identifiers explaining why that choice is blocked.

G. The engine SHALL expose a function to serialize the current game state into a snapshot.

H. The engine SHALL expose a function to restore game state from a previously serialized snapshot.

I. The engine SHALL expose a function to simulate forward from a given state using a provided decision policy, random seed, and event horizon.
